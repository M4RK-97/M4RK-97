## Filosofia ingegneristica
- Prima di rendere un sistema sofisticato, deve essere comprensibile.
- Preferisco confini espliciti, comportamento prevedibile e test dove riducono davvero il rischio.
- La manutenibilita non e un extra: naming, struttura e chiarezza operativa fanno parte del prodotto.
- Le scelte tecniche devono avere una conseguenza concreta su affidabilita, velocita di esecuzione o qualita d'uso.
